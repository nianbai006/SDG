# Where, What, Why, and Importance: Structured Defect Grounding for Text-to-Image Feedback

This repository is the official implementation of:

**Where, What, Why, and Importance: Structured Defect Grounding for Text-to-Image Feedback**

Structured Defect Grounding (SDG) formulates text-to-image diagnosis as structured set prediction, where each defect is represented as a `(location, type, reason, importance)` tuple. This project will include the SDG detector, SDG-30K dataset resources, SDG-Eval protocol, and BoxFlow-GRPO alignment code.

The code, model weights, and datasets are currently under approval and organization. We will release them as open source soon.
